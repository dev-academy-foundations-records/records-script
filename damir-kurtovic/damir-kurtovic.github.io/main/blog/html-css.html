Sprint 2 Tech - display:inline vs display:inline-block

What does it mean to display inline vs inline blocks?

CSS 'display' property defines the display behaviour of an element. An html element is a component of an html document that tells a browser how to structure itself. It consists of a start tag, a content, and an end tag. Examples of elements are the whole html document, body, heading, paragraph, etc. Html standards highly recommend using css for the elements behaviour.

There are 3 main displays in css: 1) inline, 2) block, and 3) inline-block.

In display:inline, we cannot define element's height or width, we can only include the left and right padding, it allows only left and right margins, but not top and bottom one. It automatically adjusts its height and width .As its name says with the inline display, elements are arranged in a straight line.

For an element with the block display, we can define height and width. The block element always starts on a new line. By default it occupies full line space.

Inline-block elements are arranged in a stright line, same as inline elements. But, its height and width can be defined, same as the block element. They also can have margins and paddings (same as block elements). It behaves like an image. To summarize, we use inline elements when we do not want a new line.
