How did I elegantly solved a problem

I fail a lots and achieved a lot and one by one, the last kata before strechs came through, which is 'deBuzz'. I already gained a few methods that can be used to tackle the problem. Before I started the task I wrote a pseudocode:

  1. use 'split' to break the string into array
  2. find 'buzz' out by using filter

when I reached this point, I found that I could not select and remove the words with capital words inside. I noticed that mates in Discord have discussed this before, so I went to check the reference, and it truely gave the solution, using toLowerCase() or toUperCase() to compared to buzz or BUZZ if element is not equal returning true, in this way, the code can select and remove the buzz out.

Compared to other kata challenge, I felt smooth when I tackle this 'deBuzz' problem, this was first time I guess, I could feel the flow, and it was a really nice feeling, I guess.

How confident do I feel using each of these problem-solving techniques:

  • Pesudocode

    • During this sprint, I inplement Pesudocode to all of my task, I can get most of logic right.

  • Trying something

    • I tried differnet way to tackle the same problem, and when I get blocked in a way, I tried to find out other ways like: can I use forEach to solve this problem? is there any other way that I can use? I find out the more I get stuck and the more Research I did, I am able to gain more methods that I can use to tackle different problem.

  • Rubber ducky method

    • I felt like Rubber ducky method is similar to Pseudocode, and instead of talking to a duck, I usually write down what did. It is more clear for me, as I can visualise what did I do.

  • Reading error messages

    • Every time I got my test fail, I will read the error message, and it is helpful for me to debug.

  • Console.logging

    • I would say console.log is my priority method to tackle the problem, since the Gradebook kata, I console.log almost every line to see if the code is doing what I am expecting it to do.

  • Googling

    • where I take my words about console.log back, discribe the problem and search how do I achieve this is the thing I have done thousand time a day.

  • Asking your peers for help

    • I still hinder to ask question in Discord, first I am really slow, I can see peers asked similar question in the group and usually it provides enough info for me to solve the problem.

  • Asking coaches for help

    • to be honest, I have not asked coach question privatly, I only asked few questions when having home group meeting.

  • Improving your process with reflection

    • except write the reflection provided from EDA, I sometime just do quick mind reflection, which help me to refine my way to solve the problem and start thinking and acting like a programmer.
