What is a Dom

The Dom or 'Document Object model' is a programming API for HTML and XML documents. It defines the logical structure of documents and the way a document is accessed and manipulated. In the DOM specification, the term "document" is used in the broad sense. The model itself closely resembles the structure of the documents it models. For instance, consider this table, taken from an Html document:

/Document> /HTML>
/Head> /body>

Dom Table Example

In the document Object Model, documents have a logical structure which is very much like a tree; to be more precise , it is like a 'forest' or 'grove' which can contain more than one tree. However , the Document Object Model does not specify that documents be implemented as a tree or a grove, nor does it specify how the relationships among objects be implemented in any way. In other words , the object model specifies the logical model for the programming interface, and this logical may be implemented in any way that a particular iplementation finds convenient.

How does the Dom relate to CSS and HTML?

Using the DOM in CSS in not limited to classes or ids that you create. You can use any element to walk through the document - any HTML tag in fact. Granted it is not quite as advanced as JavaScript in allowing you to move up and down the DOM but it does allow you to quickly target elements on the page. Let's say i want to make "My can of beer" bold. I don't actually need to change the document at all. Using the DOM tree i can specify a rule that will make the text bold.

The easiest way to access a single element in the DOM is by its unique ID. You can get an element by ID with the getElementById() method of the document object.

Events and it's relation to the DOM

Events are signals fired inside the browser window that notify of changes in the browser or operating system environment. Programmers can create event handler code that will run when an event fires, allowing web pages to respond appropriately to change. These 'Events' are documented in and/or below the pages for the JavaScript objects that emit them. For example these 'Events' can range from: Animation, Composition, Gestures,Inputs and Media to name a few. :) The addEventListener() method allows you to add event listeners on any HTML DOM object such as HTML elements, HTML document, the window object, or other object that support events.

What is control flow and loops?

Control flow statements let you control the flow of the execution of the code in your program. In JavaScript programming you can control the flow of execution of the code by placing the decision making , branching , looping and adding conditional blocks.

Once it is determined whether a condition is True or False the program knows what actions to take and will then continue on to the next condition to repeat the process. A anology to control flow is a kin to traffic managment.. The determine the order in wich cars are allowed to drive at every moment.

Generally we use decision-making statements when we have to decide which block of the code will be executed. There are 3 types of decision-making statements.

If statement:

The if statement is the most basic decision- making statements in the java programming language. There are two possible values of the condition , i.e. true and false. The code block after the if statement will execute only if the value of the condition statement is true. If the value of the condition statement is false, the code block will not be executed.

The if else statement is somewhat similar to the if statement, we are just adding an extra block of code after the if statement. If the value of the condtion statement is true, the if block will be executed, else else block will be executed.

The switch statement is used to perfom different actions based on different conditions. The way it works is as follows.. The switch expression is evaluated once. The value of the expression is compared with the values of each case. If there is a match, the associated block of code is executed. If there is no match , the default code block is executed.

LOOPS WHAT ARE THEY?

In programming, a loop is a sequence of instructions that is continully repeated until a certain condition is reached/ Typically, a certain process is done such as getting an item of data and changing it, and then some condition is checked such as whether a counter has reached a prescribed number.

In the for loop, we are going to check the value of the condition statement. If the value is true, the block of the code will be executed. After the successful execution of the code block, control goes to the condition statement. Now, if the value is true, again the block of the code will be executed.

A real world example of a loop would be software of the ATM machine. The ATM machine is in a loop to process transactions until you acknowledge that you have no more to do.

The difference between accessing data from arrays and objects

Objects for starters are used to represent a "thing" in your code. That could be a person, a car , a building, a book, a character in a gam. Basically anything that is made up or can be defined by a set of characteristics. In objects, these characteristics are called properties that consist of a key and a value. There are 3 ways to access data in an object. object.property, object['property'], const{ property}= object.

An array on the other hand is whenever we want to create and store a list of multiple items in a single variable. Arrays are especailly useful when creating ordered collections where items in the collection can be accessed by their numerical position in the list. The way you would access data in an array is by referring to its index number. The index arrays starts with [0], meaning the first element has index [0] and the second has [1].

What are functions and why are they helpful?

A Function is simply a "chunk" of code that you can use over and over again, rather than writing it out multiple times.

Functions are helpful because it enables programmers to break down or decompose a problem into smaller chunks, each of which performs a particular task. Once a programmer defines a function, they can then call it whenever they need it, simply using its name. However in order to work, the function will probably require some inputs or parameters, which are given to the function each time it is called.
